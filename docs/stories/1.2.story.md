# Story 1.2 — Automação de Captura de Leads

## Story

**Como** Mauro (fundador da Sparkle),
**Quero** que cada lead que entra em contato seja registrado automaticamente,
**Para que** eu nunca perca um contato e possa fazer follow-up sistemático sem depender de memória.

---

## Contexto

Hoje os leads chegam dispersos — TikTok DM, WhatsApp, indicações — e não há registro centralizado. O resultado é leads perdidos e follow-up feito de memória.

Esta story cria um fluxo n8n que captura leads automaticamente e os registra em Google Sheets. A conexão com Z-API é deixada para o final — toda a lógica é construída e testada com trigger manual primeiro, sem custo de instância.

**Dependência:** A Fase 3 (conexão Z-API) depende da Story 1.3 (Zenya Vitrine) estar concluída, pois é a vitrine que fornece o número dedicado da Sparkle. As Fases 1 e 2 podem ser executadas em paralelo com a 1.3.

**Stack:**
- n8n (fluxo de captura — já usado na Zenya)
- Google Sheets API (registro dos leads)
- Z-API (conectar por último, após tudo testado)
- Chatwoot (opcional: tag automática de novos leads)

---

### Stack existente aproveitável
- n8n já rodando no Coolify — criar novo workflow na mesma instância
- Chatwoot já tem os contatos — pode ser fonte de dados futura
- Workflow `10. Buscar ou criar contato + conversa` já faz lógica similar — estudar para reaproveitar padrão

### Estrutura do Google Sheets

```
Aba: Leads
Colunas:
A - Data/Hora (timestamp automático)
B - Nome
C - Telefone
D - Origem (WhatsApp / TikTok / Indicação / Outro)
E - Status (Novo / Contatado / Em negociação / Fechado / Perdido)
F - Próximo Passo
G - Observações
H - Data de atualização
```

### Webhook payload esperado (WhatsApp via Z-API)
```json
{
  "phone": "5511999999999",
  "name": "Nome do contato",
  "message": "primeira mensagem",
  "timestamp": "2026-03-24T10:00:00Z"
}
```

### Notificação para o Mauro
- Usar Z-API da instância existente (demo/pessoal) para enviar notificação
- Mensagem: "Novo lead: {nome} - {telefone} - Origem: {origem}"

### Armadilha: Z-API por último
- Não conectar Z-API na Fase 1 para evitar custo de instância durante desenvolvimento
- Testar tudo com webhook manual primeiro
- Só conectar Z-API quando AC1-AC4 estiverem todos aprovados
